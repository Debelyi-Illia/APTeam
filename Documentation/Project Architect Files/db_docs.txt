Файл models.py містить визначення моделей бази даних для Django-додатку. 
Він включає такі основні моделі:

1. User - Розширена модель користувача, яка додає додаткові поля, такі як справжнє ім'я, телефон, біографія тощо.
2. Subject - Модель предмету, що містить назву, опис і колірну мітку.
3. LogType - Модель, що визначає типи логів.
4. Log - Модель логування подій, пов'язаних із користувачами.
5. Advertisement - Модель для оголошень, що містять інформацію про роль, предмет, час дії та зміст.
6. Class - Модель уроку, яка зв'язує вчителя, студента, предмет і години.

Ці моделі використовуються для управління користувачами, предметами, логами та іншими аспектами додатку.

Файл db_handler.py містить клас DBHandler, який забезпечує виконання основних CRUD-операцій 
(створення, читання, оновлення, видалення) для роботи з базою даних у Django-додатку. 

Функціонал класу включає:

1. Захист від SQL-ін'єкцій через метод `check_for_injections`.
2. Перевірку прав доступу користувача за допомогою `check_user_permissions`.
3. Методи для виконання операцій з моделями бази даних:
   - `write()` – створення нового запису.
   - `read()` – читання записів із можливістю фільтрації.
   - `update()` – оновлення записів за вказаними умовами.
   - `delete()` – видалення записів.

Клас DBHandler використовується для безпечного та контрольованого управління даними в базі додатку.
