Проектувальник архітектури:
Продумати структуру проекту,
створити окремі діаграми взаємозв'язків між сторінками сайту,
створити схему БД у парі з БД+деплой розробником,
підбір алгоритмів

	
	
login->(register, home)
	обычное логин окно
register->(login, home)
	обычное регистрационное окно (почта, пароль)
alerts->(home, user, full_schedule, some_ad, class)
	оповещения: отклики людей на объявления с конкретными предложениями графиков, предупрежнения о расторжении другой стороной договоренности об установленном графике, напоминалки о начинающихся занятиях, напоминалки учителю отметить проведённое занятие если оно фактически состоялось, оповещения ученику про изменение количества проведённых и запланированных занятий, переписка пользователей, может ещё что-то. при соглашении на график предлагает отредактировать объявление на которое откликнулись/посмотреть расписание
user->(home, own_profile, search_defaults, own_ads, alerts, full_schedule)
	кабинет с тремя страничками и расписанием
own_profile->(home, user, search_defaults, own_ads, alerts)
	часть кабинета - профиль: имя и тд, контактные данные, описание о себе, прочее
search_defaults->(home, user, own_profile, own_ads, alerts)
	часть кабинета - настройки фильтров поиска по умолчанию: дни недели, часы от-до/конкретные, учитель/ученик, в общих чертах предмет уроков из списка (текст объявления, цена?)
own_ads->(home, user, search_defaults, own_profile, alerts, publish_ad, some_ad)
	часть кабинета - список своих объявлений с возможностью создать новое
publish_ad->(login, home, user, some_ad, alerts)
	оформление объявления: дата, время, учитель/ученик, название, предмет, описание. переходит к созданному объявлению. предлагает логиниться при попытке публиковать, если это не сделано
some_ad->(login, home, user, other_user, alerts)
	объявление: все прошлые параметры, плюс профиль отправителя и возможность формировать из доступных предложенных часов конкретное предложение графика занятий, который займёт слоты часов в расписании у обоих пользователей и по по которому они теоретически и будут учить/учиться
full_schedule->(login, home, user, search, class, alerts)
	расписание: табличка всех часов, дней недели, занятая, согласованными двумя сторонами и пока не расторгнутыми ни одной, графиками занятий. предлагает логиниться при попытке смотреть расписание, если это не сделано
class->(home, user, alerts)
	страница конкретного графика, учитель может задавать количество запланированных и проведённых занятий, ученик получает об этом оповещения
home->(login, home, user, search, full_schedule, alerts, publish_ad)
	стандартно слева сверху на главную страницу, справа сверху логин/кабинет пользователя, ещё рядом оповещения. сразу на главной как основное поиск и кнопки? публикации объявлений и просмотра своего расписания
search->(login, home, user, full_schedule, search_defaults, other_user, publish_ad, some_ad, alerts)
	поиск объявлений фильтрованных по: дням недели, часам от-до/конкретным, учитель/ученик, в общих чертах предмет уроков из списка, поиск по тексту (цене?). по умолчанию просто показывает свежайшие опубликованные (может ещё как-то по другому сортировать). понятно есть возможность открывать найденные объявления и профили тех кто их отправил, ещё для тех кто ничего не находит удобно сразу кнопка публикации
other_user->(login, home, user, some_ad, alerts)
	профиль с публичными данными, список объявлений пользователя, окошко отправки личных сообщений, (если вообще нечего делать отзывы какие-то?)
